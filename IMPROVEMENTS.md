## Детальное описание улучшений

### IMP-01 — Переработка логики свободного поиска (BUG-01, W-03)

**Проблема:** ИИ получал все 350 событий и сам решал, что подходит. При большом объёме данных модель путалась и возвращала нерелевантные результаты — на запрос «гончарный мастер-класс» выдавал нетворкинг-завтраки и лекции.

**Решение — двухэтапный подход:**

```
Было:
Запрос → ИИ смотрит все 350 событий → выбирает (галлюцинирует)

Стало:
Запрос → ИИ парсит критерии (теги, слова, дата, бюджет)
       → Python фильтрует базу по очкам релевантности
       → ИИ получает 3-5 готовых событий и пишет текст
```

**Система оценки релевантности:**
- Совпадение тега: +10 очков
- Ключевое слово в названии: +15 очков
- Ключевое слово в описании: +5 очков
- Тег запрошен, но у события его нет → событие исключается полностью

**Результат:** точность подбора по конкретным запросам выросла с ~40% до ~95% на тестовой выборке запросов.

### IMP-02 — Фильтрация прошедших событий (BUG-02)

**Проблема:** бот показывал события прошедших дней и прошедшие события сегодняшнего дня (пример: запрос в 23:00 → событие сегодня в 10:00).

**Решение:** добавлена функция `is_event_upcoming(event)`:

```python
def is_event_upcoming(event):
    now = datetime.now()
    event_dt = datetime.strptime(
        f"{event['date']} {event.get('time', '23:59')}", "%Y-%m-%d %H:%M"
    )
    return event_dt > now
```

Проверка применяется во всех трёх режимах поиска: кнопочный, `/ask`, `/surprise`.

**Поведение при пустой базе:** если все предстоящие события уже показаны — сбрасывается история просмотров `seen`, но прошедшие события по-прежнему не показываются.

### IMP-03 — Память диалога (BUG-03)

**Проблема:** при уточнении запроса («а подешевле?», «покажи ещё») бот начинал поиск заново, не понимая контекста.

**Решение:** хранение истории сообщений в `user_data[uid]["history"]`. При каждом запросе в GPT передаются последние 8 сообщений диалога:

```python
messages=[
    {"role": "system", "content": system_prompt},
    *u["history"][-8:],  # контекст разговора
]
```

Ассистент также фиксирует в истории какие события показал, чтобы при уточнении «покажи ещё» — не повторять их.

### IMP-04 — Персонализация (BUG-06)

**Проблема:** бот не запоминал пользователей между сессиями, каждый раз начинал с нуля.

**Решение:** при первом `/start` бот спрашивает имя и сохраняет в `user_data`. При повторном визите — приветствует по имени. Хранится также история показанных событий (`seen`) — бот не повторяет уже предложенное.

### IMP-05 — Защита FSM от неожиданных сообщений (BUG-04)

**Проблема:** если пользователь писал текст в момент выбора тегов или периода, бот молчал — состояние зависало.

**Решение:** добавлены хэндлеры для каждого FSM-состояния:

```python
@dp.message(Form.choosing_tags)
async def unexpected_in_tags(message: Message):
    await message.answer("👆 Выбери теги из кнопок выше или нажми «Готово».")

@dp.message(Form.choosing_date)
async def unexpected_in_date(message: Message):
    await message.answer("👆 Выбери период из кнопок выше.")
```

Добавлен также `fallback_handler` для любых сообщений вне состояний — направляет в главное меню.

### IMP-06 — Расширение базы событий (BUG-05, W-02)

**Проблема:** 60 событий недостаточно — при конкретных запросах (особенно мастер-классы, спорт) база давала пустые результаты.

**Решение:** база расширена до 350+ событий. Добавлен новый тег `masterclass` с 80 событиями:

| Категория | Примеры |
|-----------|---------|
| Керамика и лепка | Гончарное колесо, лепка из глины |
| Кулинария | Итальянская паста, домашний хлеб, суши |
| Искусство | Акварель, скетчинг, рисунок углём |
| Ремёсла | Макраме, свечи, украшения из смолы |
| Танцы | Танго, hip-hop для взрослых |
| Другое | Ораторское мастерство, медитация, латте-арт |

Покрытие дат: март — август 2026, равномерное распределение по дням.

### IMP-07 — Режим «Удиви меня» (W-01)

**Новая функция:** `/surprise` или кнопка 🎲 — ИИ выбирает 1-2 нестандартных события с `temperature=1.0` (максимальная неожиданность).

Особенности:
- Показывает только предстоящие события
- Не повторяет уже показанные
- При исчерпании базы — сбрасывает историю и начинает заново
