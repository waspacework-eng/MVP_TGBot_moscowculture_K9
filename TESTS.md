### 1.2 Детальный разбор критических проблем

#### СRIT-01 — Деплой Railway: бот не запускался (T-01, T-02)

Бот не запускался на Railway — два последовательных падения при первом деплое.

**Проблема 1 — отсутствие `requirements.txt`:**
```
ModuleNotFoundError: No module named 'aiogram'
Traceback (most recent call last):
  File "/app/bot.py", line 13, in <module>
    from aiogram import Bot, Dispatcher, F
```
Railway не знал какие зависимости устанавливать — файл `requirements.txt` отсутствовал в репозитории. Бот падал циклически при каждой попытке старта.

**Проблема 2 — неверный тип процесса:**
После добавления `requirements.txt` бот стартовал, но Railway запускал его как `web`-сервис и ждал открытый HTTP-порт. Бот работает через long polling и порт не открывает — Railway убивал процесс через ~30 секунд. Лог показывал многократный рестарт без явной ошибки.

**Решение:**
- Добавлен `requirements.txt` с зависимостями (`aiogram>=3.0`, `openai>=1.0.0`, `python-dotenv`)
- Добавлен `Procfile` с явным указанием типа процесса: `worker: python bot.py`


#### CRIT-02 — ИИ возвращал неструктурированный ответ (T-05)

**Проблема:** GPT-4o-mini игнорировал инструкцию возвращать JSON и отвечал произвольным текстом:
```
Вот несколько событий которые могут тебя заинтересовать! 
1. Нетворкинг-завтрак для предпринимателей — 1 апреля...
2. Митап Python в продакшне — 3 апреля...
```
Код бота пытался разобрать это как JSON -> `json.loads()` падал с исключением -> пользователь получал сообщение об ошибке вместо карточек.

**Причина:** промпт не содержал явного запрета на «обертку» вокруг JSON, и модель добавляла вступительные фразы.

**Решение:** переработан промпт — добавлены явные инструкции:
- «Верни ТОЛЬКО валидный JSON и ничего кроме него»
- Добавлена функция `strip_json()` для очистки возможных ````json` оберток
- Добавлен `try/except` с fallback на plain-text форматирование

#### CRIT-03 — ИИ выдавал нерелевантные события (T-06)

**Проблема:** при конкретных запросах ИИ подбирал события по поверхностному сходству, игнорируя смысл. Примеры зафиксированных несоответствий:

| Запрос пользователя | Что выдал ИИ |
|--------------------|--------------|
| «мастер-класс по лепке из глины» | IT-хакатон, нетворкинг-завтрак |
| «хочу на гончарный мастер-класс» | Speed networking, лекция по интернету |
| «что-нибудь творческое» | Инвестиционный форум |

**Причина:** ИИ получал все 350 событий сразу и самостоятельно выбирал — при большом объеме данных модель галлюцинировала и выбирала по случайным признакам.

**Решение:** переход на двухэтапный подход:
```
Было:  запрос -> ИИ смотрит все 350 событий -> выбирает (галлюцинирует)

Стало: запрос -> ИИ парсит критерии {теги, слова, дата, бюджет}
              -> Python фильтрует по очкам релевантности
              -> ИИ получает 3–5 готовых событий → пишет текст
```

Формат: асинхронное тестирование в Telegram. Каждый участник получил ссылку на бота и список задач. После — короткий опрос

**Задачи:**
1. Найди бесплатное событие на ближайшую неделю через кнопки

5. Оцени понятность интерфейса без инструкции
