## Часть 2. Пользовательское тестирование (фокус-группа)

### 2.1 Методология

Формат: асинхронное тестирование в мессенджере. Каждый участник получил ссылку на бота и список из 5 задач. После — короткий опрос в Google Forms.
Было принято решение остановиться на 5 модерируемых UX-тестах, поскольку такое количество достаточно для получения фидбека (85% релевантной информации).

**Задачи для тестировщиков:**

1. Оцени понятность интерфейса: смог ли ты разобраться без инструкции?
2. Найди любое бесплатное событие на ближайшие выходные
3. Найди мастер-класс (любой) через свободный запрос
4. Спроси что-нибудь и затем уточни запрос в том же диалоге

## Часть 3. Сводка найденных проблем

### Критические (блокируют ключевой сценарий)

| ID | Проблема | Где воспроизводится |
|----|----------|---------------------|
| BUG-01 | ИИ выдаёт нерелевантные события при свободном запросе | `/ask` — любой конкретный запрос |
| BUG-02 | Показываются прошедшие события | Все режимы поиска |
| BUG-03 | Бот не помнит контекст диалога при уточнении | `/ask` -> повторное сообщение |

### Значимые (ухудшают UX)

| ID | Проблема | Где воспроизводится |
|----|----------|---------------------|
| BUG-04 | Текст в режиме выбора тегов игнорируется | FSM состояние `choosing_tags` |
| BUG-05 | База событий мала — 60 записей, много дыр по датам | Поиск на конкретную дату |
| BUG-06 | Нет персонализации — бот не запоминает пользователя | `/start` повторный |

### Пожелания (nice to have)

## Выводы

Baseline выполняет базовую функцию — поиск по тегам работает стабильно. Однако ключевой сценарий (свободный текстовый запрос) работает некорректно и вызывает разочарование у пользователей. Низкая оценка точности результатов (2.4/5) — главная точка роста для следующей итерации.

Все найденные проблемы задокументированы и переданы в работу. Результаты улучшений — в файле [IMPROVEMENTS.md](IMPROVEMENTS.md).
