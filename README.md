## Описание проекта

Бот решает проблему фрагментированного доступа к информации об интересующих событиях Москвы: информация о событиях рассыпана по десяткам Telegram-каналов, сайтов и соцсетей. Пользователь тратит время на ручной поиск и часто пропускает интересное.

**Наш подход:** единая база событий + GPT-4o-mini, который понимает запросы на живом языке.

### Что умеет бот

- Принимает запросы в свободной форме: *«хочу что-нибудь бесплатное в эти выходные»*, *«ищу гончарный мастер-класс»*, *«концерт или выставка до 1000 рублей»*
- Разбирает запрос на критерии (теги, ключевые слова, бюджет, дата) и фильтрует базу программно
- Подбирает события по тегам через кнопочный интерфейс
- Предлагает неожиданные варианты в режиме «Удиви меня»
- Помнит контекст разговора — можно уточнять: *«а подешевле?»*, *«покажи ещё»*
- Не повторяет уже показанные события


## Архитектура

```
Запрос пользователя
        │
        ▼
┌───────────────────────────────────────┐
│  Шаг 1: GPT-4o-mini парсит запрос     │
│  -> теги, ключевые слова, дата, бюджет│
└───────────────┬───────────────────────┘
                │
                ▼
┌───────────────────────────────────────┐
│  Шаг 2: Python фильтрует events.json  │
│  Система оценок: тег +10, слово в     │
│  названии +15, слово в описании +5    │
│  -> топ-5 релевантных событий         │
└───────────────┬───────────────────────┘
                │
                ▼
┌───────────────────────────────────────┐
│  Шаг 3: GPT-4o-mini пишет текст       │
│  -> живое intro + крючок к каждому    │
│    событию                            │
└───────────────┬───────────────────────┘
                │
                ▼
        Красивые карточки
        пользователю
```

Ключевое решение: ИИ **не выбирает** события из полного списка (это приводит к галлюцинациям при больших базах). ИИ только **парсит запрос** и **пишет текст** – выборку делает детерминированный питон-код.


## Функциональность

### Режимы поиска

| Режим | Описание | Команда |
|-------|----------|---------|
| 🎯 По темам | Выбор тегов -> период -> подборка | `/start` -> кнопка |
| 🗣 Свободный запрос | Живой текст, ИИ разбирает | `/ask` |
| 🎲 Удиви меня | ИИ выбирает неожиданное | `/surprise` |

### Команды

| Команда | Действие |
|---------|----------|
| `/start` | Начать заново, выбор режима |
| `/ask` | Режим свободного поиска |
| `/surprise` | Случайная подборка от ИИ |
| `/find` | Поиск по сохранённым тегам |
| `/tags` | Показать выбранные интересы |
| `/help` | Справка |

### Категории событий (теги)

`education` · `tech` · `business` · `startup` · `networking` · `career` · `art` · `music` · `science` · `entertainment` · `sport` · `food` · `free` · `masterclass`

### Фильтрация по бюджету

- бесплатно
- до 500 руб.
- до 1000 руб.
- свыше 1000 руб.


## Стек технологий

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.11 |
| Telegram-фреймворк | aiogram 3.x |
| LLM | OpenAI GPT-4o-mini (API) |
| База событий | JSON (events.json, 350+ записей) |
| Деплой | Railway |
| Хранение состояний | aiogram FSM + MemoryStorage |
| Переменные окружения | python-dotenv |

> ⚠️ **Ограничение MVP:** база событий сформирована вручную – парсинг постов
> из тематических Telegram-каналов с последующей разметкой тегов. В следующей
> версии заменяется на автоматический пайплайн: KudaGo API + парсер Telegram ->
> GPT извлекает структуру -> база обновляется ежедневно

## Установка и запуск

### Локально

```bash
# 1. Клонируем репозиторий
git clone https://github.com/your-team/MVP_TGBot_moscowculture_T9
cd MVP_TGBot_moscowculture_T9

# 2. Устанавливаем зависимости
pip install -r requirements.txt

# 3. Создаём .env файл
cp .env.example .env
# Вписываем BOT_TOKEN и OPENAI_API_KEY

# 4. Запускаем
python bot.py
```

### На Railway

1. Создать новый проект в [Railway](https://railway.app/)
2. Подключить GitHub-репозиторий
3. Добавить переменные окружения (см. ниже)
4. Railway автоматически определит `Procfile` и задеплоит

## Переменные окружения

| Переменная | Описание | Обязательная |
|------------|----------|--------------|
| `BOT_TOKEN` | Токен Telegram-бота от @BotFather |
| `OPENAI_API_KEY` | API-ключ OpenAI | (для ИИ-режимов) |

> Без `OPENAI_API_KEY` бот работает в fallback-режиме: кнопочный поиск доступен, свободный запрос — нет.

### Что изменилось от baseline к финальной версии baseline

**Baseline (этап 5):**
- Поиск только по тегам через кнопки
- Фиксированные периоды дат
- 60 событий в базе
- GPT генерировал свободный текст без структуры

**Финальная версия (после тестирования):**
- Добавлен режим свободного текстового запроса `/ask`
- Двухэтапный подход: ИИ парсит -> питон фильтрует -> ИИ пишет текст
- База расширена до 350+ событий, добавлена категория `masterclass`
- Память диалога — бот помнит контекст и не повторяет события
- Персонализация — запоминает имя пользователя
- Режим «Удиви меня» с высокой температурой GPT
- Защита от галлюцинаций: программная фильтрация вместо выбора ИИ
- Честный ответ «событий нет» вместо нерелевантных результатов
